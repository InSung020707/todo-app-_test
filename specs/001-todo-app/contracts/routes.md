# Contract: Routes & Screen UI

**Date**: 2026-05-15 | **Plan**: [../plan.md](../plan.md)

App Router 경로 맵과 각 화면이 충족해야 하는 UI 계약. 시각 출력은 핸드오프 번들과
픽셀 충실(Constitution 원칙 II).

---

## Route Map

| Path | 화면 | 컴포넌트 | SideNav |
|------|------|----------|---------|
| `/` | 로그인 | `LoginScreen` | 없음 (별도 진입 화면) |
| `/main?view=<viewId>&cat=<catId>` | 메인 3단 | `MainScreen` | 있음 |
| `/calendar` | 캘린더 | `CalendarScreen` | 있음 |
| `/stats` | 통계 | `StatsScreen` | 있음 |

- `/main`의 `view` 쿼리 기본값 `today`. `cat` 쿼리가 있으면 카테고리 필터가 활성이고
  뷰 규칙보다 우선.
- `/main`에 `view`/`cat` 둘 다 없으면 `view=today`로 동작.

## SideNav 계약 (`/main`·`/calendar`·`/stats` 공유)

- 브랜드 마크 + "Tasks", 사용자 카드(아바타·이름·메일).
- 뷰 항목 5개(받은편지함/오늘/예정/지연/완료): 클릭 시 `/main?view=<id>`로 이동.
  각 항목은 `viewCount` 배지를 표시(0이면 숨김).
- "캘린더" 항목 → `/calendar`, "통계" 항목 → `/stats`.
- 카테고리 5개: 클릭 시 `/main?cat=<id>`로 이동, `catCount` 배지 표시.
- 현재 경로/쿼리에 해당하는 항목은 활성 스타일(`is-active`).
- footer: 라이트/다크 테마 토글. 접근 가능한 이름 필수.
- 모든 항목은 시맨틱 `<button>`/`<a>`, 접근 가능한 이름 보유.

## `/` — LoginScreen 계약

- 좌우 분할: 좌측 다크 히어로(브랜드, eyebrow, 타이틀, 서브, 데모 프리뷰 리스트 5행,
  푸터), 우측 폼.
- 폼: Google/GitHub 소셜 버튼, "또는 이메일로" 구분선, 이메일 입력, 비밀번호 입력
  (라벨 + "잊어버리셨나요?" 링크), "로그인 상태 유지" 체크박스, "로그인" 기본 버튼,
  하단 "무료로 시작하기" 링크.
- 로그인 버튼 또는 소셜 버튼 클릭 → `/main`(=`/main?view=today`)로 이동.
- "로그인 상태 유지" 체크박스는 토글 가능(표시 상태만, 인증 효과 없음).

## `/main` — MainScreen 계약 (3단 레이아웃)

`grid-template-columns: 248px 1fr 380px` — SideNav | TaskList | DetailPanel.

### TaskList (가운데)

- 헤더: 현재 뷰/카테고리 이름 + (오늘 뷰일 때) 날짜, 검색·더보기 아이콘 버튼(비기능).
- 필터 칩: 전체 / 우선순위 높음 / 즐겨찾기 — 현재 뷰 결과를 추가 필터.
- 추가 입력란: placeholder "할 일을 추가하세요…  Enter로 저장", Enter로 추가, 추가
  후 비워짐, 입력 중 ⏎ 키캡 표시.
- 그룹·행: `groupTasks` 결과를 그룹 라벨 + 행으로 렌더. 행은 체크박스, 우선순위 바,
  제목, 즐겨찾기 별, 메타(마감 상대표기·카테고리·서브태스크 진행·메모 아이콘).
- 표시할 행이 없으면 빈 상태 메시지.
- 행 클릭 → 해당 Task 선택(DetailPanel 갱신). 체크박스/별 클릭은 행 선택과 분리
  (이벤트 전파 차단).

### DetailPanel (우측)

- 선택 없음 → 빈 상태(아이콘 + "선택된 할 일이 없습니다").
- 선택 있음 → 헤더(완료 체크 + 진행/완료 라벨, 즐겨찾기·복제·삭제·더보기 액션),
  제목 textarea(편집 가능), 속성 그리드(마감일·우선순위·카테고리·알림), 서브태스크
  진행 막대(서브 있을 때), 서브태스크 목록 + 추가 입력, 메모 textarea.
- 우선순위·카테고리 클릭 → 인라인 팝오버, 선택 시 즉시 반영.
- 삭제 → Task 제거, 선택 해제(빈 상태로).

## `/calendar` — CalendarScreen 계약

`grid-template-columns: 248px 1fr` — SideNav | 본문.

- 헤더: 월 이름 + 연도, 이전/오늘/다음 네비, 월/주/일 토글(월만 기능, 나머지 비기능),
  "새 할 일" 버튼(비기능).
- 본문 그리드: `1fr 320px` — 월간 그리드 | 날짜 미리보기.
- 월간 그리드: 요일 헤더 7 + 42칸(6주, 일요일 시작). 각 칸은 날짜 번호 + 이벤트
  (카테고리 도트 + 제목, 최대 3개, 초과 시 "+N개 더"). 오늘·타월·일/토 구분 스타일.
- 칸 클릭 → 선택 표시 + 우측 미리보기 패널이 그 날짜 할 일 목록(없으면 빈 메시지).
- 미리보기 패널: 요일·날짜·개수 헤더, 할 일 카드 목록, "이 날에 할 일 추가" 버튼(비기능).

## `/stats` — StatsScreen 계약

`grid-template-columns: 248px 1fr` — SideNav | 본문.

- 헤더: "통계" + 기간 토글(이번 주/이번 달/분기/연간, 선택 상태만 변경).
- KPI 그리드 4칸: 완료율 / 완료한 할 일 / 연속 달성 / 가장 활발한 시간.
- 2단 패널 ×2:
  - 주간 완료 막대 차트 + 카테고리 도넛 차트.
  - 카테고리별 진행률 바 + 20주 활동 히트맵.
- 모든 수치는 `stats-data.ts`의 고정 합성값(clarify Q2).

## 비기능 컨트롤 (전 화면 공통)

검색·더보기·복제·알림·복제, 캘린더의 새 할 일/이 날에 추가, 주/일 토글, 카테고리
추가(+) 버튼 등 디자인에 존재하나 동작이 정의되지 않은 보조 컨트롤은 **시각적으로
렌더하되 비기능**으로 둔다(spec Assumptions, Constitution 원칙 III).
